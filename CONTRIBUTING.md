# CONTRIBUTING.md

## Contributing Guidelines

Thank you for your interest in contributing! Please follow these guidelines to help us maintain a high-quality codebase.

### Getting Started
- Fork the repository and create a new branch for your feature or fix.
- Install dependencies with `npm install`.
- Use `npm run dev` to start the development server.
- Run tests with `npm test` or `npm run test:unit`.

### Code Style
- Use Prettier and ESLint (run `npm run lint` before submitting).
- Write clear, descriptive commit messages.
- Use TypeScript for all new code.
- Add JSDoc comments to complex functions and components.

### Pull Requests
- Reference related issues in your PR description.
- Ensure your branch is up to date with `main` before submitting.
- Include tests for new features and bug fixes.
- Update documentation as needed (README, API_REFERENCE.md, etc).

### Feature Requests
- Open an issue to discuss new features before submitting a PR.
- Provide a clear use case and proposed solution.

### Bug Reports
- Include steps to reproduce, expected behavior, and screenshots/logs if possible.
- Label your issue appropriately (bug, enhancement, question, etc).

### Code of Conduct
- Be respectful and inclusive in all interactions.
- Report abusive or inappropriate behavior to the maintainers.

---

For questions, open an issue or contact the project maintainer.
